# 🚗 OOP Car Showroom System

## 📌 Project Overview
A console-based C# application simulating a luxury car dealership system. This mini-project was built specifically to demonstrate a strong understanding and practical application of core **Object-Oriented Programming (OOP)** principles in C#.

## 🚀 Features & OOP Concepts Applied
This system highlights the following backend development concepts:
* **Classes & Objects:** Structuring entities like `Car` and `Customer`.
* **Encapsulation:** Protecting data integrity using properties (e.g., preventing negative values for budgets and car prices using `throw new ArgumentException`).
* **Constructors:** Utilizing Default and Parameterized constructors for object initialization.
* **Deep Copying (Copy Constructor):** Implementing a custom copy constructor to safely duplicate objects containing reference types (`List<Car>`), preventing shallow copy memory leaks.
* **Method Overloading:** Creating multiple `PrintInfo` methods with different signatures to handle basic and authenticated data retrieval.

## 💻 Technologies Used
* **Language:** C#
* **Framework:** .NET Console Application

## 📌 نظرة عامة على المشروع

## 🚀 المميزات ومفاهيم الـ OOP المُطبقة
يسلط هذا النظام الضوء على مفاهيم تطوير الـ Backend التالية:
* **الفئات والكائنات (Classes & Objects):** هيكلة الكيانات مثل `Car` و `Customer`.
* **الكبسولة (Encapsulation):** حماية صحة البيانات باستخدام الخصائص (على سبيل المثال، منع القيم السالبة للميزانيات وأسعار السيارات باستخدام `throw new ArgumentException`).
* **المُشيدات (Constructors):** استخدام المُشيدات الافتراضية وذات المُعاملات لتهيئة الكائنات فور إنشائها.
* **النسخ العميق (Deep Copying):** تنفيذ مُشيد نسخ مخصص لتكرار الكائنات التي تحتوي على قوائم (`List<Car>`) بأمان، لمنع تداخل الذاكرة الناتج عن النسخ السطحي.
* **تعدد الأشكال (Method Overloading):** إنشاء عدة دوال `PrintInfo` للتعامل مع استرجاع البيانات العادي والمحمي بكلمة سر.
